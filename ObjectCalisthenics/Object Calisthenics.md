JEFF BAY 

1. Somente um nível de identação por método
2. Não use o comando else
3. Encapsule todos os tipos de dados e estruturas de dados
4. first class collections - classes que tem coleções nao tem outras responsabilidades/variaveis
5. um ponto por linha - cada linha de código deve ter apenas um ponto, ou seja, apenas uma chamada de método
6. nao abreviar os nomes
7. manter as entidades pequenas

    coesao
    testabilidade
    legibilidade
    manutencao
    evolucao do dominio

8. nao utilizar classes com mais de dois atributos que poderiam ser instancias de outras classes
9. nao utilizar getter/setters
