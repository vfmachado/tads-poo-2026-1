Pilares de POO

- Encapsulamento - Esconder os detalhes de implementação e mostrar apenas a interface para o usuário.

- Abstração - Focar nos aspectos essenciais de um objeto, ignorando os detalhes desnecessários.

- Herança - Permitir que uma classe herde características e comportamentos de outra classe.
    Classe reutiliza o código de outra classe (pai)
